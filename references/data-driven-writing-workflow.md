# 数据驱动写作工作流

公众号写作不能只看模板，也要看历史数据。

## 写稿前先看三类证据

1. 历史文章指标：阅读、分享、点赞、评论、互动率。
2. 历史正文结构：高表现文章怎么开头、怎么放截图、怎么收尾。
3. 用户当前素材：图片、prompt、截图、测试记录、用户最新修正。

## 指标读取

如果本地或 VPS 有 `wxmp-sync` 数据，优先读取：

```bash
ssh vps '/usr/local/bin/wxmp-sync export-mp --format csv --output /root/.wxmp-sync/mp_publish_metrics.csv'
```

再把 CSV 拉到当前素材包或分析目录。

如果后台 session 过期：

- 可以用已有数据库快照做第一轮分析。
- 必须标注数据可能不是最新。
- 不要声称“刚从后台抓取”。

## 爆款判断

不要只看最高阅读，也看：

- 阅读 Top
- 分享 Top
- 互动率 Top
- 同题材近似文章
- 标题和第一屏是否有强现场感

常见有效结构：

- 第一人称真实实验：`我试了...`、`我花了...`、`我把...`
- 强结果或强反差：赚钱、省钱、乱了但结果香、截图证明。
- 开头先给现场，不先讲概念。
- 中间有真实阻碍、小事故、修正。
- 结尾给个人判断，不喊口号。

## 对工具类文章的限制

工具名不能单独当卖点。必须翻译成读者能看懂的结果：

| 工具表述 | 读者表述 |
|---|---|
| Codex 连接本地工作区 | 我在手机上发消息，电脑里文件真的建好了 |
| 读取 CSV 并分析 | 它先告诉我明天该问谁 |
| 视觉理解 | 它不只说这是一张图，还能帮我选哪张能用 |
| 生成页面 | 它能先做一个能点开的第一版 |

## 写作决策

有数据时，先写一段简短判断：

```text
这篇更像哪类历史高表现文章？
它借鉴了什么？
哪些不能借鉴？
```

然后再写正文。
