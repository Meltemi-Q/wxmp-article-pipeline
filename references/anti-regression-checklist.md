# wxmp-article-pipeline 反翻车 QC 清单

用于用户说“帮我看看这篇有没有问题”“推送前检查一下”“这篇能发吗”时快速逐项核对。

输出格式固定为：

```text
QC 结论：通过 / 不通过
1. 图文对照：通过/不通过 - 证据
2. 图片使用：通过/不通过 - 证据
3. 标题封面锁定：通过/不通过 - 证据
4. 副标题 digest：通过/不通过 - 证据
5. 宇龙原话保留：通过/不通过 - 证据
6. 最近用户修正：通过/不通过 - 证据
需要改的地方：...
```

## 1. 图文对照

写完文章后必须给用户贴“图文对照表”，不是只在心里检查。

检查项：
- 每张图片都列出文件名、所在段落、图注。
- 图注说的是图片实际内容，不是文件名或泛泛描述。
- 图片和段落语义对应，没有张冠李戴。

示例：

```text
图1 (062555-ae86.jpeg) → PART 1 → 图注："库克在抖音直播间卖小米 SU7"
图2 (063012-bc91.jpeg) → PART 2 → 图注："余承东直播间卖苹果全家桶"
```

## 2. 图片默认全用

用户给的图默认全部用上，除非用户明确说“这张不用”。

检查项：
- 输入图片总数和正文使用图片数一致。
- 如果排除图片，必须能引用用户原话。
- 不要因为“看起来意义不大”擅自删图。

## 3. 标题/封面锁定

用户确认过标题或封面后禁止再优化。

检查项：
- 回复里 echo 过“已锁定：标题=...，封面=...”
- 后续推送 payload 没改标题。
- 封面 `thumb_media_id` 对应用户确认的图片。

## 4. 副标题 digest

副标题不进正文，只进 API digest，并单独发给用户。

检查项：
- `meta.json` / 正文内容不含副标题。
- `digest` 字段 ≤ 120 字。
- 回复中单独列出“副标题：...”

## 5. 优先保留宇龙原话

语音转写、口语句、用户草稿原文优先级最高。

检查项：
- 没把口语硬改成官样书面语。
- 没把“我决定换掉龙虾”改成“迁移到新平台”这种抽象话。
- 改动只做顺序调整、错字修复、段落切分。

## 6. 最近用户修正

上下文压缩、interrupt、用户说“不对/重新/应该是”之后，先读最近修正再继续。

检查项：
- 回复里复述最新确认项和修正项。
- 未继续执行旧结论。
- 如果不确定，先停下来确认，不要硬推。

## 图注口径

不要写：
- `img_04.jpg`
- `配图`
- `这是 API 调用流程图`

要写：
- `Claude Code 跑通后，终端输出 “Build successful”`
- `Hermes 在群里自动调起机票查询，把价格和航班时间列成表`
