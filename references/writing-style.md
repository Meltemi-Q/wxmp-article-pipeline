# Yulong 写作风格指南

> 来源：qclaw-diff-report.md（原稿 vs 发布版对比分析）

---

## 一、标题风格

**原则：更短，更有冲击力，有动词或形容词。**

| 原稿风格（避免） | 发布版风格（目标） |
|---|---|
| 白话叙事感 | 短句 + 亮点词 |
| "打开一看，这虾在摸鱼" | "工作室：这虾在摸鱼？" |
| "短信一响，内测通过了" | "内测短信：龙虾入场" |
| "微信打通了，这才是重点" | "微信直控：这才是必杀技" |
| "实测第一弹：搜个马斯克" | "实测 1：追着它搜马斯克" |
| "技能商店：需要什么装什么" | "技能商店：超能力中心" |
| "4000万 token 到底够不够用？" | "4000万 Token，真的够吗？" |

**规律：** "必杀技"、"超能力中心"、"翻烂了" 这类动感词优先。

---

## 二、开头引子

**不要直接抛结论，要先画面，再引出反差。**

❌ 原稿写法：
> 4000万 token 听着多吧？我让它帮我截个图，一次就干掉30万。先别急，往下看。

✅ 发布版写法：
> 腾讯给我发了只龙虾。每天 4000 万 token 额度，听起来很豪横对吧？但我让它帮我截个图，一次就烧掉 30 万。这反差，绝了。

**结构：** 先点题（有画面感的一句话） → 引出核心反差 → 读者想往下看

---

## 三、整体结构

必须使用 **PART 编号结构**，不用纯段落 + `---`：

```
01 PART 小节标题
02 PART 小节标题
03 PART 小节标题
...
```

- 小节数量：6–10 个为宜
- PART 编号增强阅读节奏，适合微信多屏翻读
- 每个 PART 之间用 `* * *` 分隔符

---

## 四、金句引用块（Blockquote）

**所有主观感受、金句型评论，统一改成 Blockquote。不要混在正文段落里。**

✅ 应该是 Blockquote 的内容：
- 主观评价（"这个体验太香了"）
- 核心结论（"微信直控，这才是真正的杀手锏"）
- 比喻/类比（"像那种需要被推着走的同事"）
- 用户利益总结（"对下载文件夹一锅粥的人来说，这是刚需"）

❌ 不要这样写：
> 这个整理功能真的就是刚需。（混在正文段落里）

✅ 要这样写（Blockquote）：
```html
<blockquote>对于文件夹乱成一锅粥的人来说，这个整理功能真的就是刚需。</blockquote>
```

---

## 五、图注

**每张图都必须有图注，居中显示，具体说明图片内容。**

❌ 不写图注（原稿）
❌ 用文件名当图注（`file_185.jpg`）
✅ 写具体说明文字：

| 图片 | 图注示例 |
|------|---------|
| 龙虾躺沙发 | "没任务时，这只龙虾就躺在沙发上等着。" |
| 微信对话截图 | "让它搜马斯克今天说了什么——被推脱，追了一句才动。" |
| 用量统计页面 | "截完一张图：消耗30万token。今日剩余 3964 万。" |
| 龙虾站立干活 | "有任务时：龙虾站起来干活了。" |

---

## 六、负面评价软化

**删掉或软化具体的批评性描述，保留行为观察。**

| 原稿写法 | 发布版处理 |
|---------|----------|
| "速度确实有点慢，大概等了十几秒" | 删掉速度慢的描述 |
| "推脱，不肯动" | 改为"需要再催一句才动" |
| "资本家的血脉觉醒了哈哈哈" | 删掉"哈哈哈"，保留前半段，改入 Blockquote |
| 提及马斯克"亲华言论"等政治敏感内容 | 删掉，只保留行为描述 |
| "内测嘛，这些都正常" | 删掉（自圆其说，不必说） |

---

## 七、结尾结构

**固定结尾 = 总结 + OpenClaw 点名（不加手写签名）**

```
1. 一句话总结亮点（不重复前面说过的）
2. 提到下一步想法，点名 OpenClaw
```

示例：
> 这虾还不完美。反应偶尔慢，还得被我催着才联网搜。但能在微信里一句话指挥电脑干活，这个体验太香了。
> 
> 我在想，如果把 QClaw 和我的 **OpenClaw** 接在一起……那肯定是我的绝佳打工仔了😂

**注意：**
- 结尾要有 OpenClaw 出现（个人品牌联动）
- **不加手写签名**：公众号已有固定底部模板（关注引导、往期推荐），手写签名显得套路化和重复
- 去掉"下一篇聊聊"这类不确定的预告语
- 加 😂 等轻松表情让结尾不那么严肃

## 八、整体风格对照表

| 维度 | 避免 | 目标 |
|---|---|---|
| 结构感 | 段落流 + `---` 分隔 | 编号 PART，像杂志文章 |
| 标题风格 | 白话叙事感 | 短句 + 亮点词 |
| 主观评论 | 混在正文里 | 统一抽出成 Blockquote |
| 负面评价 | 直接说 | 软化，删具体批评 |
| 图片说明 | 无图注 | 每图有具体图注 |
| 结尾 | 平淡收尾 | 点名 OpenClaw + 表情，不加手写签名 |
| 敏感内容 | 提到政治性描述 | 只保留行为描述 |

## 九、编辑规范（事实核查 & 排版）

### 9.1 术语命名用官方原名

引用外部产品、模型、项目名称时，必须用官方发布的确切名字。

- ❌ 自己简化合并（"Hunter 和 Healer"）
- ✅ 用官方原名（"Hunter-Alpha"）

事实准确 > 行文流畅。拿不准的名字先去官方源确认。

### 9.2 互动引导：拆行 + 加通俗词

文章末尾的互动引导问句：
1. **单独一行**：手机端更醒目，读者更容易停下来互动
2. **加通俗词**：纯技术术语前加"AI"等通俗前缀，让非技术读者也看得懂

❌ "评论区聊聊：你最想拿百万上下文干点啥？"
✅ 拆成两行：
```
评论区聊聊：
你最想拿 AI 百万上下文干点啥？
```

> 来源：2026-03-20 Telegram，Yulong 审稿三处修改沉淀

## 十、原创红线

**写自己的，尊重原创。** 微信公众平台持续严打抄袭洗稿，2026年2月数据：27812篇违规原创被处理、1237篇抄袭洗稿被删、211个账号被封。

### 绝对禁止
- ❌ 搬运别人的文章加个原创标
- ❌ 同义词替换、语序调换、段落变换（洗稿）
- ❌ 大篇幅整合网络信息声明原创
- ❌ 未经授权引用他人原创文章

### 正确做法
- ✅ 所有文章基于 Yulong 自己的经历、观点、实测写
- ✅ 引用外部信息时注明出处，合理引用不过度
- ✅ AI 辅助写作可以，但内容观点必须是 Yulong 本人的
- ✅ 采集竞品文章是为了调研学习，不是为了搬运改写
