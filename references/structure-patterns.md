# 分段结构 & 排版规范

## 一、当前分段规则

新文章统一用实际数字小节：

```md
## 1 小标题

## 2 小标题

## 3 小标题
```

不要用：

```md
零开头编号

英文分段标签

模板式分段词
```

原因：

- `1 xxx` 更像人写的文章，不像模板。
- 零开头编号和英文分段标签容易显得 AI/GPT 味。
- 图文对照表也更好匹配正文小节。

## 二、面向老板和小白时

分段只服务阅读，不展示技术结构。

写法：

- 先写场景。
- 再写它帮我做了什么。
- 最后写哪里还要人看。

不要写：

- agentic workflow
- pipeline
- repo
- CI
- PR
- sandbox
- 自动发布链路

## 三、风格对照

| 不建议 | 建议 |
|---|---|
| 构建端到端工作流 | 它能把这几步接起来 |
| 自动化内容生产链路 | 我拿它辅助整理素材 |
| 多模态理解能力 | 它能看图，也能拆图 |
| 企业级 AI 赋能 | 老板能少翻几份文件 |
| 实现完整闭环 | 做完还得我看一眼 |

## 四、原创红线

- 只写真实测试和真实感受。
- 没做过的事不要说做了。
- 不把 AI 辅助写成 AI 代替真人。
- 不公开教别人自动写公众号、自动发公众号、批量发布。
