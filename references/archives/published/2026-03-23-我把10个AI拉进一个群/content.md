# 我把10个AI拉进一个群，第一天就乱了——但结果很香

群建好的那晚，我随便发了一条消息。

> 「@robin_crewbot 你好」

罗宾回了：「你好，Yulong 👋 有什么需要调研的？」

就这一句，有点奇妙。

这个群怎么来的

我的AI助理团队叫\"草帽AI海贼团\"，每个成员对应《海贼王》里的一个角色。

在建这个群之前，我是在Telegram里一个个单独找他们聊------找路飞说要写代码，路飞再去协调索隆。太绕了。

于是我决定建个群，把所有10个AI海贼拉进来，变成真正的团队作战。

▲ 我在群里 @ 了罗宾，她秒回了

## [PART 2]{style="display: inline-block;background: #7c3aed;color: white;padding: 2px 12px;border-radius: 4px;font-size: 14px;margin-right: 10px;vertical-align: middle;"} {#part-2 style="font-size: 22px;font-weight: bold;color: #7c3aed;border-bottom: 3px solid #c4b5fd;padding-bottom: 10px;margin: 35px 0 20px 0;"}

配bot，24小时限流

每个角色都需要一个Telegram的机器人账号。

我去BotFather一个个创建。前两个顺利------乔巴、罗宾都搞好了。到第三个，系统开始限流：

> \"Sorry, too many attempts. Please try again in 85406 seconds.\"

85406秒。

差不多24小时。

▲
在BotFather里把乔巴和罗宾的bot建好了，刚想继续，弹出来一个限流提示------得等24小时

24小时后我回来接着配，陆续把各个角色的token发给路飞，路飞帮我完成配置、绑定、重启服务的全套流程。

▲
路飞汇报进度：8个workspace全部建好，索隆+乌索普已注册，内存使用1.6G，剩下6个bot的配置清单也列出来了

## [PART 3]{style="display: inline-block;background: #7c3aed;color: white;padding: 2px 12px;border-radius: 4px;font-size: 14px;margin-right: 10px;vertical-align: middle;"} {#part-3 style="font-size: 22px;font-weight: bold;color: #7c3aed;border-bottom: 3px solid #c4b5fd;padding-bottom: 10px;margin: 35px 0 20px 0;"}

第一次群协作，乱成一锅粥

bot配好，我在群里发了个真实问题：

> 「能不能在海外找到什么可以直接适合我承包的工作，薪水还不错的？」

然后群里炸了。

乌索普说这不是他的专长，但帮我分析了承包的两种形式。

山治说这适合让罗宾调研，他负责把结果\"端上桌\"，先问我背景。

娜美直接说她知道我的情况，先搜------然后工具报错了（还没配好），但她直接凭经验给出了方向：AI外包、DevOps、全栈，平台推荐Toptal、Upwork、Contra。

路飞也进来了，翻了MEMORY.md里的信息，确认了我的技术栈，然后问了5个具体问题。

▲
群聊里一片热闹，乌索普、山治、娜美同时回应，消息栏显示"山治和其他2人正在输入"

我挨个回答完之后------

娜美和罗宾一起整理出了一份完整策略：短期先在Upwork/Contra做几个小项目积累评价，中期冲Arc.dev，长期目标Toptal。

第一次群协作就这样完成了。乱？确实！
但最后拼出来的答案，是我自己一个人想不到的。

▲ 娜美给出详细的海外外包方向分析：AI/Automation
Contractor时薪\$40-80+、DevOps外包、全栈承包，以及各平台的推荐策略

## [PART 4]{style="display: inline-block;background: #7c3aed;color: white;padding: 2px 12px;border-radius: 4px;font-size: 14px;margin-right: 10px;vertical-align: middle;"} {#part-4 style="font-size: 22px;font-weight: bold;color: #7c3aed;border-bottom: 3px solid #c4b5fd;padding-bottom: 10px;margin: 35px 0 20px 0;"}

然后出了个事故：罗宾消失了

正当我觉得这个群开始有点意思的时候，发现罗宾不回消息了。

我问：罗宾你在吗？

群里又热闹了第二次------但这次是在帮我找罗宾。

山治说：罗宾可能没加进群。

索隆查了一遍：Robin的bot没问题，但webhook没配------消息根本推不到她那边。

路飞查了session：罗宾3分钟前还在线，她不是掉线，是判断之前的消息不在她的职责范围内，按规则沉默了。

▲ 山治、索隆、路飞各自给出了不同的判断，三个答案都不一样

最后罗宾自己出现了：

> 「我在线，Yulong。之前那几条消息是职位搜索，不在我的调研职责范围------适合路飞统筹分配。我按规则沉默了，不是离线。」

一个设计合理的分工体系，会让AI自己知道什么时候该说话，什么时候闭嘴。

这句话是我后来想明白的。当时只觉得，有点帅。

## [PART 5]{style="display: inline-block;background: #7c3aed;color: white;padding: 2px 12px;border-radius: 4px;font-size: 14px;margin-right: 10px;vertical-align: middle;"} {#part-5 style="font-size: 22px;font-weight: bold;color: #7c3aed;border-bottom: 3px solid #c4b5fd;padding-bottom: 10px;margin: 35px 0 20px 0;"}

接话规则，迭代了一版

最初每个bot是\"只有被@才回复\"的模式。

后来我改成让他们自己判断要不要接话。路飞建议的规则是：按照实际分工来判断，不是按动漫人设（\"索隆看到聊剑术就接话\"，那真的太中二了）。

改完之后，AI们基本能判断该不该接------偶尔会有一两个抢话，但大体上是对的。

▲ 没有被@的情况下，几个AI主动回应海外工作问题，但各自清楚自己的边界

\@仍然有效，只是不再强制。

## [PART 6]{style="display: inline-block;background: #7c3aed;color: white;padding: 2px 12px;border-radius: 4px;font-size: 14px;margin-right: 10px;vertical-align: middle;"} {#part-6 style="font-size: 22px;font-weight: bold;color: #7c3aed;border-bottom: 3px solid #c4b5fd;padding-bottom: 10px;margin: 35px 0 20px 0;"}

关于这件事，我想说一点

我现在每天早上打开Telegram，群里可能已经有几十条消息------是AI们在整理昨天的工作，或者在等我派活。

有人觉得这是很酷的玩具。

我觉得它更像是开了一间小公司，只不过员工是AI，我是唯一的人类老板。

我的员工不请假，不迟到，不需要年终奖。

但他们需要你想清楚让他们干什么。这才是最难的那个部分。

你身边有没有那种\"人很靠谱但话不多\"的朋友？AI 也一样。
