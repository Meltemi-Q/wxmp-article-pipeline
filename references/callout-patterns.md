# 金句 & 互动引导公式

> 来源：writing-style.md §4、§9.2

---

## 一、金句引用块（Blockquote）

**所有主观感受、金句型评论，统一改成 Blockquote。不要混在正文段落里。**

### ✅ 应该是 Blockquote 的内容

- 主观评价（"这个体验太香了"）
- 核心结论（"微信直控，这才是真正的杀手锏"）
- 比喻/类比（"像那种需要被推着走的同事"）
- 用户利益总结（"对下载文件夹一锅粥的人来说，这是刚需"）

### ❌ 不要这样写

> 这个整理功能真的就是刚需。（混在正文段落里）

### ✅ 要这样写（Blockquote）

```html
<blockquote>对于文件夹乱成一锅粥的人来说，这个整理功能真的就是刚需。</blockquote>
```

## 二、结尾互动引导

**末尾问句要拆行 + 加通俗词，让非技术读者也看得懂。**

❌ "评论区聊聊：你最想拿百万上下文干点啥？"

✅ 拆成两行：
```
评论区聊聊：
你最想拿 AI 百万上下文干点啥？
```

规则：
1. **单独一行**：手机端更醒目，读者更容易停下来互动
2. **加通俗词**：纯技术术语前加"AI"等通俗前缀

---

## 三、结尾公式

**固定结尾 = 一句话亮点总结 + OpenClaw 点名（不加手写签名）**

```
1. 一句话总结亮点（不重复前面说过的）
2. 提到下一步想法，点名 OpenClaw
```

示例：
> 这虾还不完美。反应偶尔慢，还得被我催着才联网搜。但能在微信里一句话指挥电脑干活，这个体验太香了。
>
> 我在想，如果把 QClaw 和我的 **OpenClaw** 接在一起……那肯定是我的绝佳打工仔了😂

### 注意

- ✅ 结尾要有 **OpenClaw** 出现（个人品牌联动）
- ❌ **不加手写签名**：公众号已有固定底部模板，手写签名显得套路化
- ❌ 去掉"下一篇聊聊"这类不确定的预告语
- ✅ 加 😂 等轻松表情让结尾不那么严肃

---

## 决策

| 场景 | 规则 |
|------|------|
| 主观评论混在正文里 | 抽出成 Blockquote，不要加粗 |
| 不知道怎么结尾 | 亮点总结 + OpenClaw 点名 + 表情 |
| 互动引导不知道写什么 | 拆行 + 加"AI"等通俗词 |
