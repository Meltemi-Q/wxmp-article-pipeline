# 图文一致性核对

发布前必须检查图片和文字是不是对得上。

## 为什么要做

公众号图文很容易出低级错：

- 正文说的是“图片拆解”，旁边放的是生成结果。
- 正文说“根据文章生成学习图”，旁边只放了文章链接截图。
- 请求截图和结果图没有成对出现。
- 图里小字看不清，但正文把小字当成关键信息。

## 核对表

写完正文后，先列这个表：

| 图片 | 正文小节 | 图里有什么 | 图里可见文字 | 正文是否对应 | 问题 |
|---|---|---|---|---|---|

每张图都要过一遍。

## 请求图和结果图要成对

这类图必须成对检查：

| 场景 | 请求图 | 结果图 |
|---|---|---|
| 参考图转风格 | 用户发起“画一个类似风格” | 生成后的风格图 |
| 文章生成学习图 | 文章链接 + 请求 | 学习图成品 |
| PPT / Word | 用户 prompt | 生成的文件预览 |

如果只有结果图，没有请求图，正文要补一句来源。

如果只有请求图，没有结果图，不能写“已经生成了”。

## 小字处理

截图里小字看不清时：

- 不把小字当成正文证据。
- 可以截局部放大。
- 或者在正文里写“这个图还需要再整理，小字太密”。

## 发布前可用 prompt

```text
帮我检查这篇文章里的图片和文字是不是对得上。

要求：
1. 每张图对应哪一段，列成表格
2. 图里能看到的文字，帮我摘出来
3. 如果正文说的事和图片不一致，直接指出来
4. 小字看不清的图，也标出来

不要帮我润色，先只做核对。
```
